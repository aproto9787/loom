Expected gain

기존 대비 개선도

아래 수치는 벤치마크가 아니라 운영 기대치입니다. low-risk 작업은 속도를 유지하고, high-risk 작업은 일부 시간이 늘어나는 대신 사고 가능성을 낮추는 구조입니다.

Before and after

원래 workflow 대비

기존

  • Claude-backed worker가 auth 상태에 막힐 수 있었습니다.
  • 위험 작업 판단이 leader prompt와 문서 규칙에 많이 의존했습니다.
  • gate, approval, rollback 상태가 run detail에서 한눈에 이어지지 않았습니다.
  • side-effect 실행 직전 차단이 실제 서버 enforcement로 고정되지 않았습니다.

적용 후

  • active runtime/default flow가 Codex-only로 정리됩니다.
  • risk tier가 manifest로 남고 Studio에서 보입니다.
  • Tier C+는 approval + rollback evidence 없이는 gate pass가 안 됩니다.
  • MCP governance tools로 worker가 상태를 직접 기록합니다.
Operating guide

적용 후 사용법

Do not

적용 시 하지 말 것

  • worktree 파일을 main checkout에 수동 복사하지 않습니다.
  • PR/CI 없이 default workflow 변경을 바로 main에 섞지 않습니다.
  • 실행 중인 오래된 Heddle MCP/server/studio를 그대로 두고 새 동작이라 판단하지 않습니다.
  • side-effect gate가 막힌 것을 문서로만 우회하지 않습니다.
